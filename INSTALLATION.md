# ⚙️ Installation Guide

Follow these steps to run the House Price Prediction System locally.

## 1. Clone the repository

```bash
git clone https://github.com/Kashaf537/HousePricePrediction_AlgoHub.git
cd HousePricePrediction_AlgoHub
```

## 2. Create a virtual environment (recommended)

```bash
python -m venv venv

# Activate it:
venv\Scripts\activate      # Windows
source venv/bin/activate   # macOS/Linux
```

## 3. Install dependencies

```bash
pip install -r requirements.txt
```

This installs: pandas, numpy, scikit-learn, matplotlib, seaborn, plotly, statsmodels, streamlit, and joblib.

## 4. Confirm required files are present

Make sure the following files are in the same folder as `housepred_app.py` before running the app:

- `house_price_model.pkl`
- `model_features.pkl`
- `feature_defaults.pkl`
- `house_data_for_viz.csv`

These are generated by running through `house_price_prediction.ipynb` end-to-end (or are already included in this repo).

## 5. Run the Streamlit app

```bash
streamlit run housepred_app.py
```

## 6. Open the app

Streamlit will print a local URL in the terminal, typically:

```
Local URL: http://localhost:8501
```

Open that link in your browser to use the app.

---

## 🔁 Running the notebook (optional)

If you want to reproduce the data cleaning, feature engineering, and model training from scratch:

```bash
jupyter notebook house_price_prediction.ipynb
```

Run all cells from top to bottom (`Kernel → Restart Kernel and Run All Cells` recommended, to avoid stale-variable issues). This regenerates all the `.pkl` files and `house_data_for_viz.csv` used by the app.

---

## 🛠️ Troubleshooting

**`FileNotFoundError` when loading `.pkl` files:**
Ensure the notebook has been run at least once, and that the generated files were copied into the same folder as `housepred_app.py`.

**`ModuleNotFoundError` for `plotly` or `statsmodels`:**
```bash
pip install plotly statsmodels
```

**Predictions look identical no matter what you input:**
Restart the Streamlit app (`Ctrl+C` then `streamlit run housepred_app.py` again) — this usually means an old cached input or stale `.pkl` file is being used.
